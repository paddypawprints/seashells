# 🐚 Seashells — Privacy-First Face Redaction Extension

A **Manifest V3** browser extension that automatically detects a specific
person's face in web page images and redacts it in real-time — entirely
on-device. No image data ever leaves your browser.

---

## Architecture

```
Browser Tab (content.js)
  │  RGBA pixel buffers (transferable ArrayBuffer)
  ▼
Web Worker (worker.js)
  ├─ ONNX Runtime Web / WebGPU EP
  │    ├─ ultraface_320.onnx  (face detection)
  │    └─ mobilefacenet.onnx  (face recognition / embedding)
  └─ Rust Wasm Core (wasm-core/pkg/)
       ├─ apply_nms()
       ├─ crop_and_normalize_batch()
       ├─ euclidean_distance()
       └─ calculate_mean_vector()

Service Worker (background.js)  — CORS proxy for cross-origin images
Options Page   (options.html)   — Training UI & settings
Popup          (popup.html)     — Enable toggle + threshold slider
```

## Build Instructions

### 1. Compile the Rust Wasm Core

```bash
cd wasm-core
wasm-pack build --target web --release
```

This produces `wasm-core/pkg/` containing:
- `seashells_wasm.js`        — JS glue (ES module)
- `seashells_wasm_bg.wasm`   — compiled Wasm binary

> **Note:** The `--target web` flag is required so the output is an ES module
> that can be imported directly from `worker.js` inside a browser extension
> (no bundler needed).

### 2. Obtain the ONNX Models

Download the two models and place them in a `models/` directory at the
extension root:

```bash
mkdir -p models

# UltraFace-320 (face detection, ~1 MB)
curl -L -o models/ultraface_320.onnx \
  "https://github.com/onnx/models/raw/main/validated/vision/body_analysis/ultraface/models/version-RFB-320.onnx"

# MobileFaceNet (face recognition, ~4 MB)
curl -L -o models/mobilefacenet.onnx \
  "https://huggingface.co/py-feat/mobilefacenet/resolve/main/mobilefacenet.onnx"
```

### 3. Obtain ONNX Runtime Web

Install `onnxruntime-web`, then copy the runtime bundle into the path expected
by `worker.js`:

```bash
npm install onnxruntime-web
./scripts/setup-onnxruntime-web.sh
```

The setup script copies the exact `.mjs` and `.wasm` files consumed by
`worker.js` into `vendor/onnxruntime-web/`.

### 4. Icons

Placeholder icons are included in `icons/`. Replace them with your own branded
16×16, 48×48, and 128×128 PNG assets if desired.

### 5. Package as a browser plugin (.zip)

After the Wasm and runtime assets are present, package the extension:

```bash
./scripts/package-browser-plugin.sh
```

This writes:
- `dist/seashells-extension/` (unpacked extension folder)
- `dist/seashells-extension.zip` (installable/distributable package)

> The full build entrypoint `./scripts/build-system.sh` now runs this packaging
> step automatically after building assets.

## Loading the Extension in Chrome / Edge

1. Navigate to `chrome://extensions` (or `edge://extensions`).
2. Enable **Developer mode** (top-right toggle).
3. Click **Load unpacked** and select the `seashells/` directory.

## Using the Extension

### Step 1 — Train the target face

1. Click the Seashells icon → **⚙️ Configure & Train**.
2. Upload 1–5 clear, front-facing photos of the person to redact.
3. Click **Train Model** and wait for the green confirmation.

### Step 2 — Enable redaction

1. Open the popup.
2. Toggle **Face Redaction** on.
3. Adjust the **Match Threshold** slider:
   - Lower (stricter) → fewer false positives.
   - Higher (more lenient) → catches more faces, risk of false positives.

### Step 3 — Configure the redaction style (optional)

In the Options page, choose:
- **Pixelate** — blurs the face region.
- **Custom Image** — overlays your own image on the face.

## Privacy Guarantee

- **No network requests** are made with image data. The background service
  worker performs only a plain HTTP `fetch` of the original image URL (for
  CORS bypass) and returns raw bytes to the content script.
- All ML inference runs inside a Web Worker in the browser tab.
- The trained face embedding is stored only in `chrome.storage.local` on your
  device.
